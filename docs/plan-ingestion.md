# Plan: Content Ingestion

## Goal

Poll RSS feeds and APIs on a schedule, fetch new articles, deduplicate, and store
in the database. This runs globally (not per-user).

## Architecture Decisions

- **Cron-triggered API route** (`/api/cron/poll`) — called by external cron (Vercel/GitHub Actions)
- **rss-parser** library for RSS/Atom feed parsing
- **Custom adapters** for non-RSS sources (HN API, HuggingFace API, Reddit JSON)
- **URL-based deduplication** — `articles.url` is UNIQUE, skip if exists
- **Batch processing** — poll all sources in one cron invocation, parallelize with Promise.allSettled
- **CRON_SECRET** header to protect the endpoint from unauthorized calls

## Data Flow

```
Cron trigger (every 2 hours)
  → POST /api/cron/poll (with CRON_SECRET header)
  → For each active source (parallel):
      → Fetch feed/API
      → Parse into normalized Article shape
      → Filter: skip if article.url already exists in DB
      → Insert new articles
      → Update source.last_polled_at
  → Return summary: { polled: 10, newArticles: 23, errors: [] }
  → Trigger AI classification for new articles (call /api/cron/classify)
```

## Source Adapters

### RSS Adapter (generic)

Used for sources 3-8, 10. Shared logic:

```
Parse feed → extract items → map to:
  { title, url, description, author, published_at }
```

### Hacker News Adapter

```
Fetch top 500 story IDs
  → Fetch details for top 50 (parallel, batch of 10)
  → Filter: score > 100
  → Map to Article shape
  → url = story URL (not HN comments URL)
```

### HuggingFace Daily Papers Adapter

```
GET /api/daily_papers
  → Map papers to Article shape
  → title = paper.title
  → url = paper URL on HuggingFace
  → description = paper.summary (AI-generated by HF)
```

### Reddit Adapter

```
GET /r/LocalLLaMA/top.json?t=day
  → Filter: score > 50
  → Map posts to Article shape
  → url = post.url (external link) or Reddit post URL if self-post
  → Requires User-Agent header
```

## Deduplication Strategy

- Primary: `articles.url` UNIQUE constraint — DB rejects duplicates
- Secondary: normalize URLs before insert (strip tracking params like utm_*, trailing slashes)
- Edge case: same article posted to multiple sources — that's fine, it shows up once
  per source but deduplicated by URL

## Error Handling

- Use `Promise.allSettled` so one failing source doesn't block others
- Log errors per source, don't crash the entire poll
- If a source fails 3 consecutive times, log a warning (don't auto-disable)
- Timeout per source fetch: 30 seconds

## Rate Limiting

- ArXiv: max 1 request per 3 seconds (they rate-limit aggressively)
- Reddit: include User-Agent header, max 10 requests/minute without OAuth
- HN: no documented limits, be reasonable (50 item fetches per poll)
- All others: standard HTTP, no special limits

## Implementation Steps

1. Install `rss-parser`
2. Create normalized `ParsedArticle` type
3. Build RSS adapter (generic, handles most sources)
4. Build HN adapter
5. Build HuggingFace adapter
6. Build Reddit adapter
7. Create `/api/cron/poll` route
8. Add URL normalization utility
9. Add CRON_SECRET validation
10. Seed `sources` table with initial 10 sources
11. Test polling locally with `curl`

## User Actions Required

### Set CRON_SECRET

Generate a random secret and add to `.env.local`:

```bash
# Generate a random secret
openssl rand -hex 32
```

Add to `.env.local`:
```
CRON_SECRET=your_generated_secret_here
```

This secret protects the `/api/cron/poll` endpoint. The cron service must send it
as a header: `Authorization: Bearer <CRON_SECRET>`.
